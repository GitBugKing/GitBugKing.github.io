North American Bike Sales Dashboard

This project involves developing a comprehensive Business Intelligence (BI) dashboard to showcase North American bike sales data from 2016 to 2024. The dashboard integrates data from multiple sources, centralized through a data warehouse, enabling efficient data analysis and reporting.

See Details

Data Sources and Architecture:

  • Database: Utilized PostgreSQL hosted on AWS RDS to store and manage bike sales data.
  • Data Warehouse: Built a data warehouse to integrate various data sources, facilitating unified data analysis and reporting.
  • ETL Process: Designed and implemented an ETL process to extract raw data into the data warehouse, perform data cleansing and transformation.
  • Data Visualization: Developed the BI dashboard using Tableau to visually present key sales metrics and trends.
  • Performance Optimization: Optimized database performance, including index optimization, query optimization, and database schema optimization to ensure fast data retrieval and reporting.

Key Insights:

Based on the data visualizations of key performance indicators (KPIs) and reports, the dashboard reveals:

  • Bike usage is higher in certain regions, correlating with local weather and seasonal sales trends.
  • The brand "Trek" and the category "road bike" show the strongest sales performance.
  • Top-performing sales representatives and most valuable customers have been identified.

Project Highlights:

  • Integration of Multiple Data Sources: The centralized data warehouse integrated data from various sources, providing a unified data view.
  • Efficient Data Visualization: Developed interactive dashboards to display KPIs, helping business users quickly understand and analyze sales data.
  • Database Performance Optimization: Enhanced database performance through index and schema optimization, significantly improving query performance and response time, supporting large-scale data processing and analysis needs.
